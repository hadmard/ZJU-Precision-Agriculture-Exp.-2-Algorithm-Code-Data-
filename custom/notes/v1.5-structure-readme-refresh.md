# 版本记录：v1.5-structure-readme-refresh

## 1. 本轮目标

本轮目标是整理项目目录结构，并重写 `README.md`，让项目更接近正式交付形态。

本轮任务类型：结构整理 + 文档增强。

## 2. 背景与上下文

上一轮已经把算法切换到 `GGA` 静态定位统计法，但项目结构仍有两个问题：

- 原始 `.nea` 文件全部散落在根目录
- `README.md` 更偏向“文件说明”，而不是“目录结构 + 使用方式 + 数学原理”

因此这轮不改算法，只做结构整理和说明重写。

## 3. 问题分析

根目录直接堆放 18 个 `.nea` 文件会带来几个问题：

- 根目录可读性差
- 不利于区分“原始输入”和“代码/结果”
- 后续增加新数据时不容易管理

同时，旧版 `README.md` 对目录结构和数学公式的强调还不够，不利于交付或给他人接手。

## 4. 候选方案比较

方案一：维持原样，只补 README。
- 优点：改动最小。
- 缺点：目录结构问题仍然存在。
- 结论：放弃。

方案二：新建专门数据目录，并同步更新 README 和脚本默认输入目录。
- 优点：目录职责清晰，项目更像正式工程。
- 缺点：需要同步更新默认路径说明。
- 结论：采用。

## 5. 最终决策

本轮最终决定：

- 新建 `data/nea/` 存放所有原始 `.nea` 文件
- 脚本默认输入目录改为 `data/nea`
- 重写根目录 `README.md`
- README 重点放在：
  - 文件夹结构
  - 核心文件职责
  - 使用方式
  - 核心算法
  - 数学原理公式

## 6. 具体实现

本轮完成了以下改动：

- 移动原始数据文件到 `data/nea/`
- 修改 `custom/nea_length/calculate_runway_distance.py`
  - 默认输入目录由 `.` 改为 `data/nea`
- 重写根目录 `README.md`
- 新增本轮版本记录 `v1.5-structure-readme-refresh.md`

## 7. 本轮优点

当前结构的优点：

- 原始数据、代码、结果、记录分区更清晰
- 根目录更整洁
- 默认运行方式更符合项目结构
- README 更适合给他人阅读和复用

## 8. 本轮缺点与代价

当前改动的代价：

- 原始文件路径发生变化，外部如果引用旧路径需要同步调整
- 部分旧说明文档如果写死了根目录 `.nea` 路径，后续需要逐步同步

## 9. 验证与结果

已完成验证：

- `.nea` 文件已成功移动到 `data/nea/`
- `python -m py_compile custom/nea_length/nea_tools.py custom/nea_length/calculate_runway_distance.py`
- `README.md` 可按 UTF-8 正常读取

## 10. 本轮结论

本轮没有改变算法结果，只是把项目结构和文档整理到了更适合继续维护和交付的状态。
