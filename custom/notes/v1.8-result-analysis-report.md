# 版本记录：v1.8-result-analysis-report

## 1. 本轮目标

本轮目标是基于当前已经固定下来的最终结果，补一份正式的结果分析报告，方便直接用于实验报告撰写。

本轮任务类型：结果分析 + 文档补充。

## 2. 背景与上下文

本轮承接：

- [v1.4-pdf-guided-static-gga.md](D:/desktop/2026精细农业探究性实验相关软件及实验报告模板/gps,bd,double result/custom/notes/v1.4-pdf-guided-static-gga.md)
- [v1.7-project-structure-image.md](D:/desktop/2026精细农业探究性实验相关软件及实验报告模板/gps,bd,double result/custom/notes/v1.7-project-structure-image.md)

截至本轮，项目已经完成：

- 原始 `.nea` 数据整理
- 端点配对规则明确
- 按 PDF 口径实现 `GGA` 静态统计法
- 结构说明和结构图文档补充

但还缺少一份专门回答“这些结果到底说明了什么”的正式分析文档。之前虽然已经有结果 CSV，但 CSV 只能给出数值，不能说明误差来源、方法差异和数据质量含义。

## 3. 问题分析

当前的核心问题不是“怎么算”，而是“怎么算完之后怎么解释”。

如果没有单独的分析文档，会出现三个问题：

- 结果表中的数值不能直接转化成实验结论
- 三种方法之间的差异缺少统一口径
- 端点质量文件虽然存在，但与最终误差之间的关系没有被明确说明

因此，本轮重点不是重跑算法，而是把已有数值解释成可以进入实验报告的文字结论。

## 4. 候选方案比较

方案一：只在 README 中追加结果说明。

- 优点：不新增文件
- 缺点：README 已经承担项目总说明职责，不适合继续堆叠详细结果分析
- 结论：不采用

方案二：只写一份简短结论，不做细分分析。

- 优点：速度快
- 缺点：无法交代误差来源、方法比较和端点稳定性关系
- 结论：不采用

方案三：新增独立结果分析报告。

- 优点：结构清晰，可直接用于实验报告引用
- 缺点：新增一个文档文件
- 结论：采用

## 5. 最终决策

本轮最终决定：

- 新增独立结果分析报告 `custom/result_analysis_report.md`
- 报告基于最终结果 CSV 和端点质量 CSV
- 不改算法、不重算数据、不改 README

## 6. 具体实现

本轮完成了以下工作：

- 读取 `custom/nea_length/runway_pair_results.csv`
- 读取 `custom/nea_length/endpoint_representatives.csv`
- 计算每组结果相对 100 m 的误差、绝对误差和相对误差
- 汇总三种方法的平均结果、平均绝对误差和最大绝对误差
- 结合端点离散度指标，分析误差与数据稳定性的关系
- 输出正式报告 `custom/result_analysis_report.md`

## 7. 本轮优点

本轮工作的优点在于：

- 把原始结果表转换成了可直接引用的实验分析文字
- 明确区分了“算法结果”和“结果解释”
- 给出了按跑道和按方法两条分析主线
- 把端点质量指标纳入结论，而不是只讨论最终距离值

## 8. 本轮缺点与代价

本轮也有明确限制：

- 报告解释基于当前结果口径，本身不提升结果精度
- 对误差原因的解释仍然属于基于现有指标的合理推断，不等于现场复测结论
- 没有引入新的图表，仅提供文字和表格分析

## 9. 验证与结果

本轮验证方式包括：

- 读取最终结果 CSV，确认 9 组结果完整
- 读取端点质量 CSV，确认每个端点的 `horizontal_std_m`、`fix_quality`、`hdop_mean` 可用于分析
- 单独计算三种方法的平均绝对误差和最大绝对误差，确认结论来源可追踪

得到的关键结论包括：

- `double` 的平均绝对误差最小，为 `14.509 m`
- `gps` 的平均结果最接近 100 m，但最大绝对误差也最大
- 东操外侧组的主要问题来自端点数据离散度过大

## 10. 本轮结论

本轮确认：当前项目已经不仅有“能跑出结果的代码”，也有“能解释结果意义的正式分析文档”。后续如果要写课程报告，可以直接基于这份报告再精简，而不需要重新从 CSV 手工归纳。
