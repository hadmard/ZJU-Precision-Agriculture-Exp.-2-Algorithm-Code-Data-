# 版本记录：v1.7-project-structure-image

## 1. 本轮目标

本轮目标是在已有 Markdown 结构图基础上，补一张可以直接插入实验报告或文档的真实图片版项目结构图。

本轮任务类型：文档补充 + 可交付性提升。

## 2. 背景与上下文

本轮承接 [v1.6-project-structure-diagram.md](D:/desktop/2026精细农业探究性实验相关软件及实验报告模板/gps,bd,double result/custom/notes/v1.6-project-structure-diagram.md)。

上一轮已经完成：

- 用 Markdown 树状图梳理了项目目录
- 把项目结构解释从 README 中抽离为独立文件

但用户进一步要求“真实图片”，这意味着仅有 Markdown 树状结构还不够，需要生成一张真实的 PNG 图片文件。

## 3. 问题分析

当前问题不是结构内容不清楚，而是交付形态不符合使用场景：

- Markdown 树状图更适合代码仓库阅读
- Word、报告、PPT 更适合直接插图
- 如果手工截图终端或 Markdown 页面，可控性差，后续也不易复用

因此，更合适的做法是生成一张程序化渲染的 PNG 图片。

## 4. 候选方案比较

方案一：继续保留 Markdown，不生成图片。

- 优点：无需新增渲染逻辑
- 缺点：不能满足“真实图片”的直接使用需求
- 结论：放弃

方案二：通过命令行内联 Python 直接生成 PNG。

- 优点：实现很快，不需要新增脚本文件
- 缺点：当前环境下 PowerShell 会把中文内联字符串转坏，图片中文字会变成问号
- 结论：放弃

方案三：新增 UTF-8 Python 渲染脚本，再生成 PNG。

- 优点：中文稳定，图片可复现，后续目录变化时也能重新生成
- 缺点：新增了一个脚本文件
- 结论：采用

## 5. 最终决策

本轮最终采用“新增 UTF-8 渲染脚本 + 输出 PNG 图片”的方案：

- 新增 `custom/render_project_structure_image.py`
- 输出图片到 `custom/project_structure_diagram.png`
- 保留原有 Markdown 结构图文件，作为图片的文本源参考

## 6. 具体实现

本轮具体改动如下：

- 新增 `custom/render_project_structure_image.py`
  - 在脚本内部直接维护 UTF-8 中文结构内容
  - 使用 Windows 中文字体 `msyh.ttc`
  - 使用 Pillow 绘制标题、目录树和说明文本
  - 输出为 PNG
- 生成 `custom/project_structure_diagram.png`
- 新增本轮记录文件 `v1.7-project-structure-image.md`

## 7. 本轮优点

当前方案的优点是：

- 直接得到可插图的真实图片文件
- 中文文本不再依赖命令行编码状态
- 后续如果目录变化，可以重新运行脚本生成新图
- 图片版和 Markdown 版同时保留，适配不同使用场景

## 8. 本轮缺点与代价

本轮的代价和限制包括：

- 新增了一个图片渲染脚本
- 图片内容不是自动扫描目录，而是按当前结构手工维护文本列表
- 后续目录变化时，脚本内容和图片都需要同步更新

## 9. 验证与结果

本轮完成的验证包括：

- 成功生成 `custom/project_structure_diagram.png`
- 实际查看图片文件，确认中文文字正常显示
- 确认没有修改算法代码和结果 CSV

## 10. 本轮结论

本轮把项目结构图从“可读的 Markdown”提升为“可直接交付的 PNG 图片”，满足了报告、PPT 和文档插图的实际需求。
